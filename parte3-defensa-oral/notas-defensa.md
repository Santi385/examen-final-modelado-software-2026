Proyecto

Nombre: Sistema de artículos
Feature principal: Guardar artículos, compartir enlaces y suscribirse a categorías.
Arquitectura: MVC.

Diseño

Estructura de clases: Cada clase tiene una función específica para mantener el sistema organizado.

MVC:

Modelo --> datos
Vista --> interfaz
Controlador --> lógica

Patrones: MVC y Observar.

Caso de Uso

Funcionalidad: Guardar artículos, compartir y recibir notificaciones.

Clases agregadas: Usuario, Artículo Compartir, Suscripción y Notificación.

Relaciones: Un usuario puede guardar, compartir y suscribirse a varias categorías.

Conceptos

Actor primario: inicia la acción.
Actor secundario: ayuda al proceso.

Precondición: lo necesario antes de ejecutar.
Postcondición: resultado final.

Clases vs Secuencia:

Clases → estructura
Secuencia → orden de acciones

SOLID: Responsabilidad única y diseño fácil de mantener.
